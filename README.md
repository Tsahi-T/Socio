# סטודיו אבחון ארגוני — Organizational Diagnosis Studio

פלטפורמת Web להנחיית תהליכי אבחון ארגוני קבוצתיים. Frontend בלבד — ללא שרת, ללא מסד נתונים, ללא הרשמה. כל המידע נשמר מקומית בדפדפן בלבד.

## מה יש בפנים

### תהליך האבחון (8 שלבים)
1. **שם הארגון** — פתיחת התהליך
2. **שמש אסוציאציות — הארגון** — רשת SNA חיה: מילים כצמתים המחוברים בקווים מונפשים לעיגול המרכזי
3. **שמש אסוציאציות — לקוחות** — איך הלקוחות תופסים אותנו (אותה רשת, פרומפט שונה)
4. **ניתוח SWOT** — חוזקות / חולשות / הזדמנויות / איומים, כולל גרירה בין רבעים
5. **במה נכון להתמקד?** — רשימת נושאים לתעדוף
6. **הצבעה** — כל משתתף בוחר בדיוק 3 נושאים; המנחה מזין את הקולות ממסך אחד; מיון חי לפי קולות
7. **יעדים אסטרטגיים** — ניסוח יעדים כשנושאי המיקוד (ממוינים לפי קולות) מוצגים בצד עם הוספה-בקליק
8. **תכלס — מטרות, מדדים ומשימות** — כל יעד נפתח באקורדיון: מטרת היעד, אופני מדידה ומשימות עיקריות

ניווט חופשי בין השלבים דרך סרגל השלבים — המנחה שולט בקצב ובסדר.

### לוח חשיבה (כלי עצמאי)
קנבס פתקים (sticky notes) למנחה: כתיבה חופשית, גרירה, שינוי גודל, 5 צבעים, קווי קשר בין פתקים, וייצוא/טעינה נפרדים.

### ייצוא וטעינה — TXT
- קובץ `TXT` קריא לאדם ולמודלי שפה, עם round-trip מלא: ייצוא → טעינה → שחזור מדויק.
- האבחון והלוח מיוצאים לקבצים נפרדים.


## ארכיטקטורה

הלב הוא **מנוע תרגילים (Exercise Engine)**: כל שלב הוא `ExerciseModule` עצמאי שנרשם ב-registry ומממש חוזה אחיד — קומפוננטת UI, יצירת מצב התחלתי, סריאליזציה ל-TXT ופענוח חזרה.

```
src/
├─ engine/        # types, registry, flow.config — ליבה אגנוסטית לתרגילים
├─ exercises/     # תרגיל = תיקייה: organization-name, association-sun,
│                 # swot, focus-topics, voting, strategic-goals, goal-breakdown
├─ state/         # SessionContext (reducer + undo + autosave), useTheme
├─ io/            # ייצוא/טעינת TXT
├─ tools/
│  └─ mindboard/  # לוח החשיבה (state, canvas, סריאליזציה משלו)
├─ ui/            # ערכת רכיבים משותפת (Button, EditableList, Stepper...)
├─ shell/         # AppShell, DiagnosisFlow
├─ theme/         # design tokens — בהיר/כהה
└─ i18n/          # מחרוזות עברית מרוכזות
```

**הוספת תרגיל חדש** = תיקייה חדשה תחת `exercises/` + רישום ב-`exercises/index.ts` + שורה ב-`flow.config.ts`. אפס שינוי בליבה.

## עקרונות
- **פרטיות מלאה** — אין רשת, אין telemetry, אין cookies; הנתונים ב-localStorage בלבד
- **עמידות** — שמירה אוטומטית, שחזור אחרי רענון, Undo, אישור לפני פעולות הרסניות, טיפול בקבצים פגומים
- **RTL מלא + עברית**, מצב כהה, Responsive, אנימציות עדינות (150–250ms)
- **מוכנות ל-AI** — קובץ הייצוא בנוי להזנה ישירה למודל שפה
